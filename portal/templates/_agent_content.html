Hướng dẫn Thêm Agent

Các agent giám sát (Collector Agent) được thiết kế để tự động kết nối và gửi dữ liệu đến ObsEngine khi được triển khai trên môi trường mục tiêu.

Để thêm một agent mới, bạn cần triển khai nó trên môi trường đó (ví dụ: Kubernetes cluster, máy ảo Linux, Windows Server,...).

Các bước triển khai chung:

  1. Chuẩn bị gói cài đặt hoặc manifest phù hợp với nền tảng (ví dụ: manifest K8s, script cài đặt cho VM).
  2. Cấu hình các thông số cần thiết cho agent, đặc biệt là:
    • OBS_ENGINE_URL: Địa chỉ của ObsEngine API này (bắt buộc).
    • AGENT_ID: Tên định danh duy nhất cho agent này (ví dụ: agent-k8s-prod-01, agent-linux-web-01).
    • ENVIRONMENT_NAME: Tên định danh cho môi trường mà agent đang chạy (ví dụ: k8s-prod-cluster, webserver-01).
    • ENVIRONMENT_TYPE: Loại môi trường (kubernetes, linux, windows, docker...).
    • Thông tin kết nối đến nguồn log (ví dụ: LOKI_URL cho Kubernetes, đường dẫn file log cho VM).
    • (Tùy chọn) Cấu hình ban đầu cho các tham số như tần suất quét, mức log,... (có thể cấu hình lại qua giao diện sau).
  3. Thực hiện triển khai agent trên môi trường mục tiêu (ví dụ: `kubectl apply`, chạy script cài đặt).

Sau khi triển khai và khởi chạy thành công, agent sẽ tự động kết nối và xuất hiện trong danh sách "Trạng thái Agent" ở trên.

Lưu ý: Chức năng tạo agent trực tiếp từ giao diện này chưa được hỗ trợ. Vui lòng tham khảo tài liệu triển khai chi tiết cho từng loại nền tảng.
