Hướng dẫn Thêm Agent

Các agent giám sát (Collector Agent) được thiết kế để tự động kết nối và gửi dữ liệu đến ObsEngine khi được triển khai trên môi trường mục tiêu.

Để thêm một agent mới, bạn cần triển khai nó trên môi trường đó (ví dụ: Kubernetes cluster, máy ảo Linux, Windows Server,...).

Các bước triển khai chung:

  1. Chuẩn bị gói cài đặt hoặc manifest phù hợp với nền tảng (ví dụ: manifest K8s, file binary `obsagent` cho Linux).
  2. Cấu hình các thông số cần thiết cho agent:
    • **Đối với K8s Agent:** Sử dụng ConfigMap/Secret để đặt biến môi trường (`OBS_ENGINE_URL`, `AGENT_ID`, `K8S_CLUSTER_NAME`, `LOKI_URL`,...).
    • **Đối với Linux Agent:** Tạo file cấu hình `obsengine.cfg` (ví dụ: trong `/etc/obsagent/`) với các thông số `ObsEngineURL`, `AgentID`, `EnvironmentName`, và các tùy chọn giám sát khác.
  3. Thực hiện triển khai agent trên môi trường mục tiêu (ví dụ: `kubectl apply`, chạy file binary `obsagent` dưới dạng service systemd).

Sau khi triển khai và khởi chạy thành công, agent sẽ tự động kết nối và xuất hiện trong danh sách "Trạng thái Agent" ở trên.

Lưu ý: Chức năng tạo agent trực tiếp từ giao diện này chưa được hỗ trợ. Vui lòng tham khảo tài liệu triển khai chi tiết cho từng loại nền tảng.
